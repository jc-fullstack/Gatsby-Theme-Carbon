<PageDescription>

The `<Video>` component can render a Vimeo player or a html video player.

</PageDescription>

## Example

<Title>Vimeo</Title>

<Video title="Carbon homepage video" vimeoId="359578263" />

<Title>Video</Title>

<Video src="/videos/hero-video.mp4" poster="/images/poster.png">
  <track kind="captions" default src="/videos/vtt/hero-video.vtt" srcLang="en" />
</Video>

<Title>Video in MDX</Title>

<Video src={localVideo} poster={localPoster} />

## Code

<Title>Vimeo</Title>

```jsx path=components/Video/Video.js src=https://github.com/carbon-design-system/gatsby-theme-carbon/tree/master/packages/gatsby-theme-carbon/src/components/Video
<Video title="Carbon homepage video" vimeoId="322021187" />
```

<Title>Video</Title>

```jsx path=components/Video/Video.js src=https://github.com/carbon-design-system/gatsby-theme-carbon/tree/master/packages/gatsby-theme-carbon/src/components/Video
<Video src="/videos/hero-video.mp4" poster="/images/poster.png">
```

<Title>Video in MDX</Title>

```markdown path=components/Video.mdx src=https://github.com/carbon-design-system/gatsby-theme-carbon/blob/master/packages/example/src/pages/components/Video/index.mdx
import localVideo from './local-video.mp4';
import localPoster from './local-poster.png';

<Video src={localVideo} poster={localPoster} />
```

#### Note to those using the video in MDX with Github Pages

If you are trying to implement the video inside an MDX file and are hosting with Github Pages you may run into an issue where during local development the video loads properly, but once pushed to Github Pages the video fails to load due to a 404 error. One known cause is due to the usage of `pathPrefix` inside your `gatsby-config.js` file. To fix this it is recommended to put a `static` folder in the root of the repository, place the video file in there, and then instead of directly importing the video file into your MDX file like in example above you would then simply implement like this `<Video src="/video.mp4" />`
